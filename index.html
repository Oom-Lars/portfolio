PROJECT 01Flagship · .NET + React · Solo

Fulfilment Returns — a new autonomous returns flow

I solo-built this end-to-end, rewriting a heavily manual Delphi returns process into a modern, largely autonomous flow. An agent searches a customer number, picks an order, and the screen auto-filters to eligible orders — showing the return eligibility period, with a management override I lifted from the Delphi logic and rebuilt in .NET. From there they choose a reason and method (refund or replace, each with its own backend ruleset) and a drop-off point (a HomeChoice store or any Pargo location). A summary is sent to our third-party partner, which integrates with the couriers and returns a waybill to initiate the return. I built the C# / .NET backend methods over our on-prem Omega SQL systems and used AI to help build the React front end against them — and owned the gnarly parts, like an EF navigation bug that silently skipped inserts and stalled the "Receive Waybill" step.

SIDE 01Product · .NET + React

chkin.co.za — Digital Patient Intake

I built the framework and system behind chkin.co.za for its owner using AI, who's since taken it further and onboarded clients. It takes the pain out of seeing a doctor: instead of filling in the same paperwork over and over, you enter your details once in the app or website. Scan a QR code at a practice and it pulls up that practitioner's form, cross-references it against your data, and fills it in automatically. It also covers visit statistics, a saved history of your completed forms, and consent management between patients and their doctors — and lets practitioners build their own forms with a QR code generated on the app. Built with React + TypeScript and a C# / .NET backend on MySQL.
